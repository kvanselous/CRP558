Tech Report: ArcGIS JavaScript API

Introduction:

ArcGIS JavaScript API is similar to Leaflet as they are both tools developers use to build maps, embed maps in webpages, and connect to libraries to develop maps (Teixeira, 2020). ArcGIS JavaScript API differs from Leaflet in that it allows developers to build 3D visualizations, conduct additional geospatial analysis, and share data easily across Esri products (Teixeira, 2020). Because ArcGIS JavaScript API works across Esri products, it allows creators to pull data from ArcGIS Online or ArcGIS servers to build maps. Additionally, it allows creators to integrate ArcGIS widgets across Esri products to increase efficiency (Powell, 2016). The newest version is 4.14 and includes improved capabilities such as the addition of label options for 3D features, a daylight widget which allows creators to display shadows and lightening in 3D scenes, a widget that allows creators to examine the 'line of sight' from various points in a 3D scene, the ability to update and add attachments to feature layers, tools that help the layer compare visualizations over time periods, and improved clustering abilities that includes GeoJSON layers (Powell & Svensson, 2019). To learn more about developers review of this product please click here, here, and here. Please review the below to gain more information about ArcGIS online, APIs, and JavaScript.

What is ArcGIS online?
ArcGIS online is a communication platform that allows users to map location data and connect it with a variety of information. This tool is used by a wide array of industries including private companies, government, and non-profits. It is also easy to share. Not only is it a tool that can that can help organizations drive data informed decision making, but it is also a tool that can help visualize data in interesting and easily interpreted ways (Esri, 2015; Esri, 2020a). This tool can also be used to store and share data that is used to build maps in ArgGIS JavaScript API. What is an API?
An API, or Application Programming Interface, is often described as similar to a waiter in a restaurant (MuleSoft Videos, 2015; Simply Explained, 2019). In other words, it is a tool that transitions data and information between computer applications, similar to the way a waiter delivers food to a table from the kitchen. Commonly used applications, such as the weather app or cheap airfare applications like Kayak, use APIs to connect data to the user by way of the application. Four key purposes of the APIs, as identified in the linked resource in this section, include support in accessing data, hiding website complexities for developers, extending functionality of systems, and acting as gatekeepers to protect user's personal data (Simply Explained, 2019). These are important aspects to keep in mind as the API allows developers to use minimal and consistent code when creating data visualizations with maps. For more information and examples of APIs can be found in the references here and here.

While Leaflet and ArcGIS JavaScript API both use APIs, ArcGIS JavaScript API has more features than Leaflet. Take a look at the feature page for Leaflet and ArcGIS JavaScript API. You'll notice that the Leaflet page emphasizes its focus on basic features. Some of the features we've used in previous coursework include visual features to change our maps such as the zoom and pan map options, as well as layer controls (Leaflet, 2019). You'll notice these features are included in the ArcGIS API reference guide and that this tool includes features for additional analysis such as features that calculate area measurement, conduct time series animation, and conduct 'line of sight' analysis (Esri, 2020b).

Overall Review

Overall, this tool appears to be a 'step up' from Leaflet due to its increased API capabilities and tools for analysis. While Leaflet is often praised for its simplicity, it seems to be largely used to create maps with basic functionality. On the other hand, ArcGIS JavaScript API demonstrates more advanced features, like the 3D visualization feature, which provides creators with levels of analysis that are not accessible in Leaflet.

Not only does this provide map creators with additional tools to display data but the controllers in the top left hand of the screen allow the viewer to explore data in a way that fosters increased engagement. For example, the map above shows polygons outlines of building structures in Aspen, Colorado. The navigation buttons in the left of the screen allow the user to navigate through this regionâ€™s elevation and see how building structures correspond with varying elevations in the region. While this information can be displayed on a 2D map, the 3D allows the viewer to explore a new visualization and perspective that allows the user to interpret the data differently than if displayed on a 2D map.

Another feature that is unique to this tool is the way ArcGIS JavaScript API connects with other Esri products. I explored the ArcGIS Developer Dashboard as part of this tutorial which connects ArcGIS JavaScript API and ArcGIS Online. While I found this tool extremely helpful, I did find it difficult to navigate initially. Similarly, I found the tutorials difficult to digest at first. These initial difficulties may have been more representative of my overall beginner level understanding of coding and web development. As a result, I would recommend beginner coders stick with Leaflet until they feel more comfortable with general coding before advancing to this tool.

Conclusion

In conclusion, I would recommend this tool for intermediate to advanced developers or those interested in expanding beyond the basic functionality of Leaflet. I would also recommend this tool for beginners that are motivated to display maps with 3D scenes or within time lapse animation, as this is a unique feature to this map, although it may take more time for beginner coders to feel comfortable with this tool. Similarly, those that are interested in building maps and sharing data across teams may also benefit from transition to this tool due to its sharing capabilities.
